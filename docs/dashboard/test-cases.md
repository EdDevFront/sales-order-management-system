# Casos de Teste - Dashboard (Painel de Monitoramento Operacional)

Este documento define os cenários de teste E2E para validação das regras de negócio associadas ao Painel de Monitoramento Operacional (Dashboard), incluindo métricas, filtros dinâmicos e tabela paginada de ordens de venda.

---

## 🔴 Casos de Teste Críticos

| ID / Cenário                                                 | Objetivo                                                                                                                       | Pré-condições                                                                                        | Passos                                                                                                                                    | Resultado Esperado                                                                                                                                                                                                                                                                                                                                 |
| :----------------------------------------------------------- | :----------------------------------------------------------------------------------------------------------------------------- | :--------------------------------------------------------------------------------------------------- | :---------------------------------------------------------------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **CT-DASH-01**<br>Carregamento inicial com métricas corretas | Validar que as 4 métricas operacionais são exibidas com os valores corretos ao carregar o Dashboard.                           | - Pelo menos 1 pedido em cada status (CRIADA, PLANEJADA, EM_TRANSPORTE, ENTREGUE) no localStorage.   | 1. Acessar a rota `/dashboard`.<br>2. Aguardar o carregamento dos dados (skeleton some).                                                  | - Card "Total de Pedidos" exibe a soma total de `salesOrders` no banco.<br>- Card "Requer Agendamento" exibe a contagem de pedidos com status `PLANEJADA`.<br>- Card "Em Transporte" exibe a contagem de pedidos com status `EM_TRANSPORTE`.<br>- Card "Entregues" exibe a contagem de pedidos com status `ENTREGUE`.<br>- Nenhum erro no console. |
| **CT-DASH-02**<br>Filtro combinado por Status e Cliente      | Validar que a aplicação simultânea dos filtros de Status e Cliente retorna apenas os pedidos que atendem a AMBOS os critérios. | - Existem pedidos do cliente "Acme Logistics SA" (`cust-1`) em diferentes status.                    | 1. No filtro "Status", selecionar "Criada".<br>2. No filtro "Cliente", selecionar "Acme Logistics SA".<br>3. Clicar em "Aplicar Filtros". | - A tabela exibe apenas pedidos com status `CRIADA` E cliente `cust-1`.<br>- O contador de paginação reflete o total filtrado.<br>- O botão "Limpar filtros" fica visível.                                                                                                                                                                         |
| **CT-DASH-03**<br>Limpeza de todos os filtros ativos         | Validar que o botão "Limpar filtros" restaura todos os pedidos e reseta a página para 1.                                       | - Filtros de Status e Cliente estão ativos.<br>- A tabela está na página 2 dos resultados filtrados. | 1. Clicar em "Limpar filtros".                                                                                                            | - Todos os selects de filtro voltam para "Todos os Status" / "Todos os Clientes" / "Todos os Transportes".<br>- O DatePicker é limpo.<br>- A tabela exibe todos os pedidos sem filtro.<br>- A paginação volta para a página 1.<br>- O botão "Limpar filtros" desaparece.                                                                           |
| **CT-DASH-04**<br>Filtro por Data de Criação                 | Validar que o filtro de data retorna apenas pedidos criados na data selecionada.                                               | - Existem pedidos com `createdAt` em datas diferentes no banco.                                      | 1. Selecionar uma data específica no DatePicker.<br>2. Clicar em "Aplicar Filtros".                                                       | - A tabela exibe apenas pedidos cujo `createdAt` começa com a string da data selecionada (formato YYYY-MM-DD).<br>- Se nenhum pedido corresponder, exibe o estado vazio com "Nenhum resultado corresponde aos filtros".                                                                                                                            |
