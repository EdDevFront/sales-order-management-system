# Casos de Teste - Ordens de Venda (Orders)

Este documento define os cenários de teste E2E para validação das regras de negócio associadas ao fluxo de criação, transição de status, agendamento e gestão de ordens de venda.

---

## 🔴 Casos de Teste Críticos

| ID / Cenário                                                       | Objetivo                                                                                                 | Pré-condições                                                                                                                                                                                          | Passos                                                                                                                                                                                                                                                                                              | Resultado Esperado                                                                                                                                                                                                                                            |
| :----------------------------------------------------------------- | :------------------------------------------------------------------------------------------------------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **CT-ORD-01**<br>Criar ordem de venda com sucesso                  | Validar a criação completa de uma nova ordem de venda vinculando cliente, transporte autorizado e itens. | - Cliente "Acme Logistics SA" (`cust-1`) está cadastrado com transportes autorizados: Caminhão e Carreta.<br>- Itens "Heavy Duty Engine Part" (`item-1`) e "Steel Support Beam 6m" (`item-2`) existem. | 1. Clicar em "Novo Pedido".<br>2. Selecionar Cliente = "Acme Logistics SA (CNPJ)".<br>3. Selecionar Transporte = "Caminhão".<br>4. Adicionar Item "Heavy Duty Engine Part" com quantidade 2.<br>5. Adicionar Item "Steel Support Beam 6m" com quantidade 1.<br>6. Clicar em "Criar Ordem de Venda". | - Modal fecha.<br>- Toast "Ordem de Venda criada com sucesso!" é exibido.<br>- Novo pedido aparece no topo da tabela com status "Criada".<br>- Total do pedido = (1500 × 2) + (450 × 1) = R$ 3.450,00.<br>- Um registro de auditoria `CREATE_ORDER` é gerado. |
| **CT-ORD-02**<br>Validação de transporte não autorizado na criação | Validar que o sistema REJEITA a criação de pedido com tipo de transporte não autorizado para o cliente.  | - Cliente "John Doe Distribution" (`cust-2`) só tem "Caminhão" autorizado.                                                                                                                             | 1. Clicar em "Novo Pedido".<br>2. Selecionar Cliente = "John Doe Distribution (CPF)".<br>3. Tentar selecionar Transporte = "Carreta" ou "Bi-truck" (não devem aparecer no select).                                                                                                                  | - O select de Transporte exibe APENAS "Caminhão" como opção.<br>- "Carreta" e "Bi-truck" NÃO aparecem na lista.                                                                                                                                               |
| **CT-ORD-03**<br>Transição de status CRIADA → PLANEJADA            | Validar a transição de status com confirmação visual e registro de auditoria.                            | - Existe pedido `so-1` com status "Criada".                                                                                                                                                            | 1. Na tabela, clicar em "Gerenciar" no pedido `so-1`.<br>2. No painel lateral, clicar em "Transicionar para Planejada".<br>3. Na tela de confirmação, clicar em "Confirmar Transição".                                                                                                              | - Modal de confirmação exibe: "Criada → Planejada".<br>- Status do pedido muda para "Planejada" na tabela.<br>- Toast "Status atualizado para Planejada!" é exibido.<br>- Registro de auditoria `UPDATE_STATUS` com previousState e nextState é criado.       |
| **CT-ORD-04**<br>Agendamento de entrega (PLANEJADA → AGENDADA)     | Validar o fluxo completo de agendamento de entrega para pedido no status Planejada.                      | - Existe pedido com status "Planejada".                                                                                                                                                                | 1. Clicar em "Gerenciar" no pedido planejado.<br>2. Clicar em "Agendar Entrega".<br>3. Selecionar uma data futura no DatePicker.<br>4. Selecionar janela "Manhã (08:00 - 12:00)".<br>5. Clicar em "Confirmar Agendamento".                                                                          | - Status do pedido muda para "Agendada".<br>- Data e janela de entrega são exibidas no painel de detalhes.<br>- Registro de auditoria `UPDATE_DELIVERY` é criado.                                                                                             |
| **CT-ORD-05**<br>Transição AGENDADA → EM_TRANSPORTE → ENTREGUE     | Validar o fluxo completo até a entrega final.                                                            | - Existe pedido com status "Agendada".                                                                                                                                                                 | 1. Clicar em "Transicionar para Em Transporte" → Confirmar.<br>2. Verificar que status muda para "Em Transporte".<br>3. Clicar em "Transicionar para Entregue" → Confirmar.                                                                                                                         | - Status final = "Entregue".<br>- OrderStepper exibe todos os passos anteriores como concluídos (check verde).<br>- Nenhum botão de transição adicional é exibido (fluxo concluído).<br>- Mensagem "Fluxo concluído para este pedido." é exibida.             |

| **CT-ORD-06**<br>Bloqueio de transição inválida de status           | Validar que o sistema REJEITA transições fora da ordem correta (ex: pular de CRIADA direto para AGENDADA). | - Pedido `so-1` está com status "Criada".                                                                                   | 1. Via console/devtools, tentar disparar `updateStatusRequest` com `newStatus: "AGENDADA"`.                                                                         | - Saga retorna erro: "Invalid status transition from CRIADA to AGENDADA".<br>- Toast de erro é exibido com a mensagem.<br>- Status do pedido permanece "Criada".                              |
| **CT-ORD-07**<br>Edição de tipo de transporte no painel de detalhes | Validar a troca do tipo de transporte para pedidos nos status CRIADA ou PLANEJADA.                         | - Pedido está com status "Criada" e transporte "Caminhão".<br>- Cliente tem "Carreta" como transporte autorizado adicional. | 1. Abrir painel de detalhes do pedido.<br>2. No select "Tipo de Transporte", trocar de "Caminhão" para "Carreta".                                                   | - Transporte é alterado imediatamente (sem confirmação adicional).<br>- Registro de auditoria `UPDATE_TRANSPORT` é criado.<br>- Toast "Tipo de transporte atualizado com sucesso!" é exibido. |
| **CT-ORD-08**<br>Transporte bloqueado para edição após PLANEJADA    | Validar que o select de transporte fica readonly para pedidos com status AGENDADA ou superior.             | - Pedido está com status "Agendada".                                                                                        | 1. Abrir painel de detalhes do pedido.                                                                                                                              | - O tipo de transporte é exibido como texto (não é um select editável).<br>- Nenhum dropdown aparece para alterar o transporte.                                                               |
| **CT-ORD-09**<br>Reagendamento de entrega (AGENDADA/EM_TRANSPORTE)  | Validar que é possível reagendar a data e janela para pedidos já agendados.                                | - Pedido está com status "Agendada" e data de entrega definida.                                                             | 1. Abrir painel de detalhes.<br>2. Clicar no ícone de lápis (✏️) ao lado dos detalhes de entrega.<br>3. Selecionar nova data e janela.<br>4. Confirmar agendamento. | - Data e janela são atualizadas.<br>- Status permanece "Agendada".<br>- Registro de auditoria `UPDATE_DELIVERY` com previousState/nextState é criado.                                         |
| **CT-ORD-10**<br>Criação de pedido sem itens (validação)            | Validar que o formulário REJEITA a submissão sem adicionar pelo menos 1 item.                              | - Modal "Criar Ordem de Venda" aberto.                                                                                      | 1. Selecionar cliente e transporte.<br>2. Remover todos os itens da lista.<br>3. Tentar submeter o formulário.                                                      | - Mensagem de erro "Adicione pelo menos um item" é exibida.<br>- O formulário NÃO é enviado.                                                                                                  |
| **CT-ORD-11**<br>Cálculo do total do pedido na tabela               | Validar que o total exibido na coluna "Total" da tabela de pedidos está correto.                           | - Pedido `so-1` tem itens: item-1 (R$ 1.500,00 × 2) + item-3 (R$ 120,00 × 5).                                               | 1. Visualizar a tabela de pedidos.                                                                                                                                  | - Coluna "Total" do pedido `so-1` exibe: R$ 3.600,00 (1.500×2 + 120×5).                                                                                                                       |
